Sentiment Analyzer Web App

Fast sentiment analysis with resilient fallback logic for reliable user experience.

Read Full Documentation View Source Code

What It Solves

API-driven sentiment features often degrade when upstream services are slow or unavailable. This project combines remote inference with local fallback and request-level optimizations to keep responses usable and fast.

Highlights

  • Flask backend with browser UI for text sentiment analysis
  • Remote-first inference with graceful local fallback
  • Blank-input handling with explicit user guidance
  • Connection reuse, cooldown logic, and cache-based performance improvements

Architecture

  1. User submits text in the UI.
  2. Frontend sends a request to /sentimentAnalyzer.
  3. Backend validates input and checks blank text early.
  4. Analyzer attempts remote model inference.
  5. If unavailable, local fallback returns sentiment safely.
